Development Methodologies:

  • Design driven
  • Prototype driven
  • Domain driven (DDD)
  • Test driven (TDD)

Design driven (Waterfall)

Design driven (Waterfall)

  • Based on the civil engineering disciplines
  • Splits design from implementation
  • Requirments are agreed at the start of the project
  • Bugs found during the design stage are cheaper to fix

Prototype driven

Prototype driven

  • Rapid application development
  • Release the product as soon as possible
  • Requirments are re-examined after each release

Domain driven

Domain driven

  • Focus on the core domain and domain logic
  • Design based on a model of the domain
  • Collaboration between technical and domain experts

Test driven

Test driven

  • Write tests before product code
  • Refine interfaces with use case mocks
  • Quickly uncover edge cases and pitfalls

Workflow: General Outline

  1. Plan
  2. Create
  3. Verify
  4. Staging
  5. Production
  6. Configure
  7. Monitor

Workflow: Test Driven Development

  1. Write a failing test
  2. Right just enough code to make the test pass
  3. Refactor the code without breaking the test
  4. Repeat

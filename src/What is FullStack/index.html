It is a qualifier for people who work on mostly all aspects of a system.
E.g. She is a full stack electronics engineer.
He is a full stack web developer.
Are there different types of full stack?
Yes, as many as types of systems.
Electronics Systems would require understanding chips, sensors
Cryptographic systems require mathematics and threat modeling
Operating systems - computer architectures (cpu, memory, etc)
Games - 3d, game theory, networking
There are also many overlapping topics.
This course focuses on Full Stack Web Development.
Which technologies are full stack?
There are many technology stacks that provide utilities on almost all levels. However being Full Stack does NOT mean knowing a set of technologies. It means knowing the layers and the principles by which they operate and connect.
Are there any general full stack principles?
  • Quality
  • Automation
  • Testing
  • Monitoring
  • Agile Processes
Quality

create automated test suits on multiple levels (unit, integration, acceptence) in order to catch problems as early as possible

What are the web layers and principles?
There are, basically, three layers: Front-end, Back-end and the Network. However each layer has its own ecosystem of tools and best practices. In {{Web Architectures}} we will explore typical patterns found in popular web applications. We will also look into the {{RESTful principles}} and the {{SOLID principles}}.

Frontend

{{Frontend Overview}} | {{Frontend Tooling}} | {{Frontend Frameworks}}
Can be a static page in a browser, an ajax -, or a mobile app. Your code runs on the user's device with restricted previliages. In all cases, the frontend is the user faceing side of the system. The input/output mechanism of the web application.

Backend

{{Backend Overview}} | {{Backend Tooling}}
Can be a home pc, data center rack or a virtual container in the cloud. On the backend your code is king. It is where the complex, secret, and long-running tasks should run. The backend is the motherboard of the web application (with processing, memory and storage attached).

Network

{{Network Overview}}
The network connects the front-end, the back-end and the WORLD. The internet is based on the TCP/IP protocol. The web is based on the HTTP protocol. Other popular protocols include: HTTPS, WebRTC, WebSockets, BitTorrent, etc OSI stack Multiple modes of communication: request/response, streaming, Multiple degrees of durability, reliability, security, latency and through-put
What is a typical dayly workflow for a full stack developer?
  • Analysing domain specific processes and data.
  • Writing code, tests, automation scripts and documentation.
  • Creating monitoring graphics, dashboards and alerts for both the application and the underlying infrastructure.
  • Analysing the current state of both the code-base and the running application.
  • Fix bugs, improve performance and usability.
